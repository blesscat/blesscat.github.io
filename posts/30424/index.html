React.js vs Vue.js

Posted on 2018-09-04 | Edited on 2018-09-10 |

前言

其實標題有點誇大為了騙點擊
只是剛好接到一個小題目，要求要使用React，有一陣子沒有碰了
剛好溫習一下

再加上最近vue用的挺熟
這兩個框架又是目前最熱門的angular表示:
剛好就我的感想來說說兩者的差異

對我來說兩邊都有我覺得非常好的功能，很難說出哪個比較好用
只有哪個比較上手而已

PS: 這裡比較在不包含Vuex 與 Redux 的情況下

從專案建立開始

React 有 create-react-app
Vue 有 vue-cli (3.0版本最近才出，不是很熟，以下還是用2.x版本作比較)

##React

create-react-app創立的專案非常簡潔，常用的功能都包在react-scripts 裡面，像是webpack JSX Autoprefixed babel 等等…

package.json的dependencies只有3個

習慣用的scss要另外裝，官方有教學

webpack設定全部都藏在node_modules裡面，身爲一個孳生攻城獅當然想要會修改些更符合自己的設定
網路上查到的大部分都是教使用npm run eject ，這樣所有檔案都會移到專案資料夾(官方不建議這樣使用)

後來發現一個專案叫react-app-rewired
只要在根目錄加上一個config-overrides.js檔案，將要修改的設定直接寫入即可
舉例要加上alias只要這樣即可

vue預設有個很好的動畫方式<transition-group>
react也找到個有個套件叫做react-transition-group，試用了一下用法跟vue的用法非常像
但因為沒有使用到太多功能，不方便評論

ESLint好像也要另外安裝，這點不是很確定因為這次案子我沒有裝

Vue

上面有提到以下比較主要還是以2.x版本為主，因為3.0版本大改不是很熟
但就我所認識的是跟react-create-app越來越相似

vue-cli製作專案時有很多樣本可以使用

用法很簡單

安裝中也會提供一些選項供選擇，例如要不要使用vue-route，npm or yarn，要不要用ESLint，用哪種規則( Standatd、 Airbnb、 none)，要不要test runner，使用哪套(Jest、 Karma and Mocha、 none)等等

在安裝時就可以選擇並且幫你都裝好，不太需要自己另外再另外安裝
我個人使用時是在安裝好後再修改.eslintrc.js，增刪一些自己習慣的coding style
就可以開始寫APP了

Summary

就建立專案來說，說實話因為我比較熟vue ，用vue-cli也開啟過不少案子，相對的會熟非常多也是一定的
但就我這次測試來說，我個人還是認為vue建立專案會比react簡單快速很多
